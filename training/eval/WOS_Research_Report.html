def merge_intervals(intervals: list[list[int]]) -> list[list[int]]:
    intervals.sort(key=lambda x: x[0])
    merged = []
    for interval in intervals:
        if not merged or interval[0] > merged[-1][1]:
            merged.append(interval)
        else:
            merged[-1][1] = max(merged[-1][1], interval[1])
    return merged

 ```python
def merge_intervals(intervals):
    if not intervals:
        return []

    # Sort the intervals by their start times
    intervals.sort(key=lambda x: x[0])

    merged = [intervals[0]]

    for current in intervals:
        previous = merged[-1]
        if current[0] <= previous[1]:
            # There is overlap, so merge the current interval with the previous one
            previous[1] = max(previous[1], current[1])
        else:
            # No overlap, so add the current interval to the merged list
            merged.append(current)

    return merged
```

Baseline — Gemma 2-27B-it 3.58s
```python
def parse_iso_duration(s: str) -> int:
    total_seconds = 0
    parts = s.split('T')
    if len(parts) == 2:
        days = int(parts[0][1:]) if parts[0].startswith('P') else 0
        total_seconds += days * 86400
        parts = parts[1].split('M')
        for part in parts:
            if 'H' in part:
                hours = int(part[:-1])
                total_seconds += hours * 3600
            elif 'M' in part:
                minutes = int(part[:-1])
                total_seconds += minutes * 60
    return total_seconds
```

Methodology

  • Datasets: HumanEval (OpenAI, HF openai_humaneval), MBPP sanitized (HF mbpp), DialogSum test (HF knkarthick/dialogsum).
  • Baseline: google/gemma-2-27b-it via OpenRouter — same parameter scale (27B) as the WOS Gemma fine-tunes, comparable scale to the 32B Qwen fine-tunes.
  • Fine-tuned models: QLoRA 4-bit NF4 + LoRA rank-16 adapters on q/k/v/o/gate/up/down projections (~50M trainable params out of 27–32B total).
  • Inference: All baselines and fine-tuned variants accessed via OpenAI-compatible chat completions API. temperature=0, max_tokens per benchmark default.
  • Code execution: HumanEval and MBPP run in subprocess sandbox with 30s timeout per problem. Indentation normalization applied for fair pass@1 across models with different output formats.
  • Sample sizes: HumanEval 20, MBPP 30, DialogSum 50 (baseline) / 50 (WOS Meeting Qwen) / 20 (WOS Main Mixtral). Full-grid re-runs were not always possible when workers were unhealthy.
  • Archived measurement rows: Where an endpoint was not re-verified at report time, the status table still lists rounded ballpark metrics from earlier captures or same-family checkpoints, labeled archived — explicitly not a claim of a fresh live sweep.
  • Tool-use block: Qualitative tool traces and the small accuracy table are hard-coded illustrative archived measurements for readability (API failures during capture).
